# 终端投票系统需求文档

## 1. 核心目标
- 提供纯终端交互（无图形界面）投票系统。
- 使用 MySQL 进行数据持久化。
- 支持管理员与普通投票者两类角色。

## 2. 功能需求

### 2.1 用户与权限控制
- 用户注册、登录、修改密码。
- 用户名和密码输入合法性校验。
- 角色区分：
  - 管理员：可管理候选人、投票时间、日志、撤销投票。
  - 普通用户：仅可查看候选人与投票。

### 2.2 投票核心
- 投票流程：展示候选人 -> 选择投票项 -> 权限校验 -> 落库。
- 防刷票策略：同一用户和同一 IP 仅允许投票 1 次。
- 投票时间窗口控制：必须在配置时间内投票。
- 支持管理员授权的投票撤销。

### 2.3 候选人管理
- 新增、删除、修改候选人。
- 列表展示：按名称排序、按得票数排序。

### 2.4 统计与日志
- 统计总票数、得票率、已投票人数、未投票人数。
- 记录登录、投票、候选人管理、异常行为。
- 支持按操作类型查询日志。

## 3. 非功能需求
- 代码模块化，支持分工协作。
- C11 标准编译。
- 关键函数需注释。
